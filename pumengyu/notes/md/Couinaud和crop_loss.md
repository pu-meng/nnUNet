# 研究方向分析

## 一、Couinaud 肝段分割的可能性

### 1.1 临床价值：真实且巨大

FLR 评估 + 8段自动划分确实是外科领域刚需，目前手工操作耗时，自动化价值极高。这个方向发 TMI / MIA 完全没问题，临床 motivation 无可挑剔。

### 1.2 现有数据的局限

Couinaud 分段的边界不是灰度边界，是由**血管树**定义的：

| 分段 | 定义血管 |
|------|----------|
| 第一段（尾状叶） | 肝短静脉 |
| 第二～四段 | 肝左静脉 + 门静脉左支 |
| 第五～八段 | 肝右静脉 + 肝中静脉 + 门静脉右支 |

做这个**必须**有：
- 肝静脉标注
- 门静脉标注（至少到 2-3 级分支）
- 理想情况下还要有动脉

LiTS / MSD Task03 只有 liver + tumor 两个标签，没有任何血管标注，131 个 case 数据对这个问题完全无效，需要从零换数据集。

### 1.3 有血管标注的公开数据集

| 数据集 | 标注内容 | 规模 | 问题 |
|--------|----------|------|------|
| 3D-IRCADb | 肝脏+肿瘤+肝静脉+门静脉 | 20 cases | 太小，5-fold CV 根本不够 |
| TotalSegmentator | 多器官含部分血管 | 1200+ | 无 Couinaud 段标注 |
| MSD Task08 (Hepatic Vessel) | 肝脏血管+肿瘤 | 303 | 无段标注，血管是二值 mask |
| 专科医院私有数据 | 完整标注 | — | 需要合作 |

**核心问题**：没有一个公开数据集有现成的 8 段标注。自己标注的话，一个 case 需要放射科医生花 2-4 小时，131 个 case 是 260-500 小时标注量。

### 1.4 结论

| 维度 | 评估 |
|------|------|
| 临床价值 | ★★★★★ |
| 论文创新性 | ★★★★☆ |
| 对你现有数据/实验的继承性 | ★☆☆☆☆（基本从零开始） |
| 毕业时间线内可行性 | **极低**（数据获取就要半年以上） |

这个方向本身是好方向，但和现在做的东西没有衔接，换方向的代价接近重新开题。

### 1.5 更务实的建议

你手里已经有：
- 完整的 5-fold 实验数据
- TwoStage 框架
- 量化出来的 train-test gap（2.77 点）

**Crop-Consistency Loss** 那个方向直接用现有数据做，3 个月内出结果，投 MICCAI 时间上来得及。Couinaud 那个方向可以写进 Future Work 或者博士论文的后半段，配合临床合作单位来做。

## 二、Crop-Consistency Loss

### 2.1 是什么

同一个 case（比如 `liver_120`）的两条推理路径：

**训练时的现状：**
```
GT 肝脏框裁剪 → Stage2 → 预测肿瘤 → Dice Loss → 更新参数
（推理时框变了，模型不认识，Dice 掉了 0.33）
```

**加了 Consistency Loss 之后：**
```
GT 框裁剪       → Stage2 → 预测A ──┐
                                    ├─ 一致性Loss（预测A和B要接近）
Stage1预测框裁剪 → Stage2 → 预测B ──┘

整体Loss = Dice(预测A, GT标签) + λ × MSE(预测A, 预测B)
```

**核心思想**：同一个病人的肿瘤就在那里，不管框从哪里裁，Stage2 给出的预测应该一致。训练时强制它一致，推理时碰到偏一点的框就不慌了。

### 2.2 能做吗？

可以做，但有一个前置工程量。

**已经有的东西：**

| 已有 | 作用 |
|------|------|
| Dataset004（GT crop 预处理数据） | 正常训练路径，不用动 |
| `cache_stage1.py` | 已经能批量生成 Stage1 预测的肝脏 mask |
| `eval.py` 里的裁剪逻辑 | 已经实现了"预测 mask → 裁剪 CT"的完整代码 |
| Dataset003 的 Stage1 模型权重 | 直接用，不用重训 |

**唯一需要新做的**：把 131 个训练 case 的 Stage1 预测框裁剪图，按 nnUNet 格式预处理好，存成"平行数据集"，训练时配对读取。

### 2.3 最大的技术难点

两个裁剪图大小不一样：

```
GT 框裁剪后的图    → nnUNet 预处理 → 比如 [220, 340, 370] voxels
Stage1预测框裁剪后 → nnUNet 预处理 → 比如 [215, 336, 368] voxels（略小）
```

直接逐体素对比行不通，需要对齐到同一坐标系。

**解决方案有两个：**

- **方案一（简单）**：不对齐体素，在特征层做一致性。两条路的特征向量维度相同，直接 MSE，不需要空间对应。
- **方案二（更严谨）**：两条路都 resize 到同一个 patch size（nnUNet 本来就会 resize），在输出层做一致性，但只计算 GT 框和 Stage1 框空间重叠区域内的 loss。

方案一实现简单很多，论文里可以先做方案一，消融实验里比较两者。

### 2.4 实现步骤

**Step 1 — 生成 Stage1 预测裁剪数据（约 1-2 天工作量）**
- 用已有 `cache_stage1.py` + `eval.py` 里的裁剪逻辑，离线生成所有训练 case 的 S1 predicted crop
- 存到 `nnUNet_workspace/raw/Dataset004_LiverTumor_S1crop/`，nnUNet plan+preprocess 一遍

**Step 2 — 写新 Trainer（约 2-3 天）**
- 重写 `get_tr_and_val_datasets`，同时加载 GT crop 数据集和 S1 crop 数据集
- 重写 `train_step`，两条路各 forward 一次，加 consistency loss

**Step 3 — 跑实验对比**
- 训练时间约 57h/5-fold，和现在相同

### 2.5 Contribution

> 现有两阶段方法要么用 GT crop 训练（有 gap），要么用预测 crop 训练（Stage1 误差污染训练集）。
> 本文提出的 **Crop-Consistency Loss** 同时利用两种 crop，用一致性约束在不丢弃 GT 监督信号的前提下消除分布偏移。

**直接量化对比**：`nnUNetTrainer_TwoStage` E2E Dice 0.7396 → 加了之后提升多少，这个 Δ 就是你的实验结论。
