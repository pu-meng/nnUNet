## 三、方法总评

### 3.1 方法有效性边界

| 方法 | 解决了什么 | 解决不了什么 | 副作用 |
|------|-----------|-----------|--------|
| **UFL** | 中大肿瘤的 loss 不平衡（liver_74 +0.54） | A/B/C 类全部无效 | 无肿瘤误报率不变 |
| **CopyPaste** | 极微小肿瘤（liver_83 +0.59，唯一方法） | 其他极小 case 无效 | 无肿瘤误报率 33%→100% |
| **TwoStage** | 中大肿瘤搜索空间缩小（4例解决） | A/B/C 类，且引发退步 | liver_29/35 严重退步 |
| **BndLoss** | liver_74/77（与TwoStage重叠） | 大部分难例 | 不稳定（liver_25 退步） |

### 3.2 不可消除的 Performance Floor

```
A类（物理不可解）:  4 例
B类（数据质量）  :  2 例
C类（体积极小）  :  2 例
─────────────────────────
绝对 floor        :  8 例 / 118例（6.8%）
如排除 floor 后均值 Dice 估算: 待计算
```

## 六、寻求导师建议的问题

1. **难例处理策略**：A类（等密度）8例是物理上界，是否应在论文中明确提出并排除这 8 例后报告"可解案例集"上的性能？

2. **liver_95 的意义**：全实验 Dice=0 的唯一案例，建议在方法局限性分析中单独展示。如何在论文中适当叙述？

3. **CopyPaste 的取舍**：极微小肿瘤方向唯一有效的方法（liver_83 0.19→0.78），但副作用严重（无肿瘤误报100%）。加后处理是否足够，还是需要根本上改变粘贴策略？

4. **UFL 5-fold 均值预期**：fold_1/4 均有 +3.4pp 提升，如果 5-fold 均值稳定在 +2pp 以上，这个量级够不够 MICCAI 的主贡献？

5. **论文框架**：建议以"难例分层分析"作为分析章节的核心，展示现有方法的边界——您认为这个角度是否新颖足够？
