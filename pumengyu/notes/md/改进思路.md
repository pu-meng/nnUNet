Ladaptive = α(x)·LDice+(1−α(x))·LCE, α(x) = exp(−U (x))
这个的损失函数,U(x)是通过推理多次dropout的方差或者,可以看作是每一个体素的困难程度,

## 第二阶段：改动中等，收益显著

### 4. Size-stratified Loss
**类别**：Loss 设计  
**论文**：2025 arxiv 系列  
**原理**：按目标体积大小分层计算 loss 权重，体积越小的目标自动获得越高的惩罚系数：
```python
weight = 1.0 / (target_volume_voxels + epsilon)
```
**实现注意**：nnUNet 以 patch 为单位训练，需在 `train_step` 内对当前 patch 的 GT 动态估计肿瘤体积，比 per-scan 计算复杂，改动成本高于 Unified Focal Loss。  
**与 Unified Focal Loss 的关系**：可叠加，Focal Loss 处理 voxel 级不平衡，Size-stratified 处理 case 级不平衡。

---

### 5. U-Mamba（混合 CNN-SSM 架构）
**类别**：网络架构  
**论文**：*U-Mamba: Enhancing Long-range Dependency for Biomedical Image Segmentation*，arXiv 2401.04722，2024  
**作者**：Jun Ma, Bo Wang（Vector Institute，Toronto）  
**代码**：https://github.com/bowang-lab/U-Mamba  

**核心原理**：  
Mamba 基于状态空间模型（SSM），复杂度为 O(N)，相比 Transformer 的 O(N²) 可以在**高分辨率特征图**上建模全局依赖。

```
每个 U-Mamba Block：
  ResBlock（卷积，局部特征）
+ ResBlock（卷积，局部特征）
+ Mamba Block（全局长程依赖，线性复杂度）
```

**对小肿瘤的意义**：小肿瘤在 128³、64³ 的高分辨率特征图里还存在，Mamba 在此建模其与周围血管/边界的全局关系，而 Transformer 在这个分辨率下显存爆炸跑不起来。

**与 nnUNet 的兼容性**：U-Mamba 直接继承 nnUNet 自配置机制，代码结构高度兼容，替换 encoder block 即可。

**实践建议**：
```
第1层（最高分辨率）：只用 ResBlock（显存压力大）
第2–4层：U-Mamba block（分辨率适中，收益最大）
Bottleneck：U-Mamba block（全局上下文）
```

---

### 6. Large Kernel Attention (LKA)
**类别**：网络架构（轻量注意力）  
**论文**：*Visual Attention Network (VAN)*，CVPR 2023  
**原理**：将大卷积核（如 21×21）分解为三步：
```
深度可分离卷积 + 深度可分离空洞卷积 + 1×1 卷积
```
模拟大感受野 attention，但计算量远小于 self-attention。空洞卷积不做下采样，不损失分辨率，小肿瘤的局部纹理得以保留。  
**插入位置**：encoder 第 2–3 层（高分辨率层，小肿瘤还存在的地方）。

---

### 7. Overlap-aware Loss
**类别**：Loss 设计  
**论文**：MICCAI 2024  
**原理**：传统 Dice Loss 中，小目标区域容易被大目标或背景的预测"压制"，梯度贡献极小。Overlap-aware Loss 在计算时识别预测重叠区域，给被压制的小目标额外的梯度权重。

---

### 8. TTT-Seg（测试时训练）
**类别**：训练策略  
**论文**：*Test Time Training for Medical Image Segmentation*，2024  
**原理**：在推理阶段，利用自监督信号对模型做短暂梯度更新（几个 gradient step），使模型权重适配当前测试样本的分布。  

> ⚠️ **注意**：TTT-Seg **不是 nnUNet 内置功能**。nnUNet 内置的是 **TTA（测试时增强）**，即推理时做镜像翻转然后对预测结果取平均——TTA 不修改模型权重。TTT-Seg 需要在推理时执行反向传播，需要**单独实现**，改动成本比表中标注的"中"更接近"高"。

**对本任务的意义**：LiTS（训练）→ 3D-IRCADb（外部验证）存在域偏移，TTT-Seg 可以改善跨数据集泛化，对小肿瘤跨域检测尤其有帮助。建议在内部 5-fold 结果稳定后再考虑实现。

## 附：Crop-Consistency Loss 设计方案

> 针对两阶段 train-test gap（GT crop 验证比 E2E 高估 2.77pp）的专项解法。

### 问题

TwoStage 训练用 GT bbox 裁剪（完美边界），推理用 Stage1 预测 bbox 裁剪（边界有偏差）。
训练-推理分布不一致，导致 E2E Dice 系统性低于 GT-crop 验证 Dice。

### 方案

同一 case 走两条路：

```
GT 框裁剪       → Stage2 → 预测A ──┐
                                    ├─ 一致性 Loss（A 和 B 要接近）
Stage1预测框裁剪 → Stage2 → 预测B ──┘

总 Loss = Dice(预测A, GT标签) + λ × MSE(预测A, 预测B)
```

核心思想：同一病人的肿瘤不管框从哪裁，Stage2 给出的预测应该一致。训练时强制一致，推理时对不完美的框更鲁棒。

### 技术难点与解法

两条路裁剪图大小不同，无法逐体素对齐：

| 方案 | 做法 | 难度 |
|------|------|------|
| 方案一（推荐先做） | 在特征层做一致性（MSE on feature vectors，不需要空间对应） | 低 |
| 方案二 | 两路 resize 到同一 patch size，在输出层只算空间重叠区域的 loss | 中 |

### 实现步骤

1. 用已有 `cache_stage1.py` + `eval.py` 裁剪逻辑，离线生成所有训练 case 的 Stage1 预测 crop，存为平行数据集（约 1-2 天）
2. 新 Trainer：重写 `get_tr_and_val_datasets` 同时加载两个数据集，重写 `train_step` 做两次 forward（约 2-3 天）
3. 跑 fold_0/4 对比实验，量化 Δ Dice

### Contribution 表述

> 现有两阶段方法要么用 GT crop（有 gap），要么用预测 crop（Stage1 误差污染训练集）。
> 本方法同时利用两种 crop，用一致性约束在不丢弃 GT 监督信号的前提下消除分布偏移。
