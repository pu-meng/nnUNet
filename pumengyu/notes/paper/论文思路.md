### 第三步：论文创新模式 + 潜在方向调研

**医学图像分割论文的主流创新模式：**

| 模式 | 描述 | 适合程度 |
|------|------|---------|
| 新模块插入现有网络 | 识别UNet某环节缺陷 → 设计模块替换/增强 → 插入nnUNet → 消融实验 | ★★★★★ |
| 新损失函数 | 现有loss有X缺陷 → 提出新loss → 标准网络上对比实验 | ★★★★★ |
| 新训练策略 | 课程学习、难样本挖掘，不改网络只改训练过程（如oversampling逻辑、采样顺序） | ★★★ |
| 新架构（Transformer/Mamba变体） | 已饱和，竞争极大，**已排除** | ✗ |

**适合自己的论文故事线：**

```
已知问题（文献承认）
    → 现有方法（含nnUNet）在该问题上的具体缺陷
    → 提出新模块 / 新loss
    → 插入nnUNet，消融 + 对比实验
    → 结论
```

> nnUNet 作为 backbone，不重新设计网络结构。单卡GPU跑不起大架构实验。
> 训练策略（课程学习、难样本挖掘）可作为辅助实验，**不是主要创新点**。主要创新点在损失函数设计。

**训练策略说明（区别于调参和数据增强）：**
- 数据增强：对图像做随机变换（翻转/旋转），解决过拟合
- 训练策略：控制训练过程本身——采什么样本、什么顺序、权重怎么动态调整
- 例：Foreground Oversampling（保证每个batch含前景patch）、Hard Sample Mining（优先采预测最差的样本）、Curriculum Learning（先易后难）

## 论文逻辑链（师哥建议）

```
看论文找到已被承认的问题
    → 分析现有方法在该问题上的不足
    → 提出针对性解决方案
    → 用实验证明有效
    → 自己的数据分析作为动机佐证
```

**不要**从自己数据里发现问题再找方案——样本量不够，统计上无法自证。
