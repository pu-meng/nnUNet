## 研究背景

**任务**：CT 图像中肝脏 + 肝脏肿瘤联合分割（liver=1, tumor=2）。

### 数据集

| 数据集 | 说明 |
|--------|------|
| 训练：MSD Task03_Liver（LiTS） | 131 cases，5-fold CV；肿瘤阳性 96 例，无肿瘤 35 例；门脉期增强 CT，512×512 |
| 外部测试：3D-IRCADb | 20 cases，DICOM 格式，用于验证跨数据集泛化 |

### Baseline 结果（nnUNet 3d_fullres，5-fold）

| Fold | Liver Dice | Tumor Dice | 有肿瘤 n |
|------|-----------|-----------|---------|
| fold_0 | 0.9652 | 0.7530 | 24 |
| fold_1 | 0.9602 | 0.6257 | 23 |
| fold_2 | 0.9509 | 0.6791 | 26 |
| fold_3 | 0.9572 | 0.6933 | 26 |
| fold_4 | 0.9505 | 0.6021 | 23 |
| **avg** | **0.9568** | **0.6706** | 96 |

- fold_0 最好（0.753），fold_4 最差（0.602），差距 0.15
- **开发策略**：日常迭代只跑 fold_0 + fold_4，确认方向后再跑完整 5-fold

### 完全失败 Case（文献佐证用）

| case | 根本原因 |
|------|---------|
| liver_39 | contrast=0.0 HU，肿瘤与肝脏 HU 完全一致，物理无解 |
| liver_116 | contrast=-8.9 HU，等密度大肿瘤（61万体素），完全漏检 |
| liver_43 | 肝脏 HU=28.7（疑似非增强CT），数据质量异常 |
| liver_127 | 肿瘤只有 298 体素，体积极小 |

> 注：失败案例只有相关性，不足以自证因果，作为动机佐证即可，不作为 motivation 主体。

### 第三步：论文创新模式 + 潜在方向调研

**医学图像分割论文的主流创新模式：**

| 模式 | 描述 | 适合程度 |
|------|------|---------|
| 新模块插入现有网络 | 识别UNet某环节缺陷 → 设计模块替换/增强 → 插入nnUNet → 消融实验 | ★★★★★ |
| 新损失函数 | 现有loss有X缺陷 → 提出新loss → 标准网络上对比实验 | ★★★★★ |
| 新训练策略 | 课程学习、难样本挖掘，不改网络只改训练过程（如oversampling逻辑、采样顺序） | ★★★ |
| 新架构（Transformer/Mamba变体） | 已饱和，竞争极大，**已排除** | ✗ |

**适合自己的论文故事线：**

```
已知问题（文献承认）
    → 现有方法（含nnUNet）在该问题上的具体缺陷
    → 提出新模块 / 新loss
    → 插入nnUNet，消融 + 对比实验
    → 结论
```

> nnUNet 作为 backbone，不重新设计网络结构。单卡GPU跑不起大架构实验。
> 训练策略（课程学习、难样本挖掘）可作为辅助实验，**不是主要创新点**。主要创新点在损失函数设计。

**训练策略说明（区别于调参和数据增强）：**
- 数据增强：对图像做随机变换（翻转/旋转），解决过拟合
- 训练策略：控制训练过程本身——采什么样本、什么顺序、权重怎么动态调整
- 例：Foreground Oversampling（保证每个batch含前景patch）、Hard Sample Mining（优先采预测最差的样本）、Curriculum Learning（先易后难）

**需要调研的代表论文（主攻损失函数设计，看高引论文的Method写法）：**

## 论文逻辑链（师哥建议）

```
看论文找到已被承认的问题
    → 分析现有方法在该问题上的不足
    → 提出针对性解决方案
    → 用实验证明有效
    → 自己的数据分析作为动机佐证
```

**不要**从自己数据里发现问题再找方案——样本量不够，统计上无法自证。
